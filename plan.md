# Шаг 1: Дерево в массиве (30 минут)
Разберись, как замапить дерево на обычный `int arr[]`.
- Для узла с индексом `i`:
  - Левый потомок: `2 * i + 1`
  - Правый потомок: `2 * i + 2`
  - Родитель: `(i - 1) / 2`
- Что попрактиковать: Напиши обход ЛКП для массива. Если элемент равен условной заглушке (например, `-1` или `INT_MIN`), значит узла нет.
# Шаг 2: Графы — Код Дейкстры и Флойда (45 минут)
Тебе нужно уметь писать их с закрытыми глазами. Дейкстру на матрице за $O(V^2)$ написать проще всего, зазубри этот шаблон. Флойда-Уоршела зазубривать не надо — это буквально три вложенных цикла `for (k)`, `for (i)`, `for (j)`.
- Специфика задач: Обрати внимание на формулировку «путь, проходящий через заданное ребро $(u, v)$». Решается элементарно: `кратчайший_путь(start -> u) + вес(u, v) + кратчайший_путь(v -> end)`. Тебе просто нужно запустить Дейкстру дважды (от `start` и от `v`), либо взять готовые данные из матрицы Флойда.
# Шаг 3: Синтаксический анализ и Стек (30 минут)
Повтори алгоритм «Сортировочной станции» Дейкстры для инфиксной $\to$ постфиксной записи и вычисление по постфиксной записи. Напиши код вычисления по ОПН:
- Идешь по строке. Видишь число — пушишь в стек. Видишь оператор — достаешь два числа из стека, применяешь оператор, результат пушишь обратно.
