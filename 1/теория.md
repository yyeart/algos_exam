# Сортировка Хоара (быстрая сортировка)

## Алгоритм
Выбирается опорный элемент (pivot). Массив разделяется на две части: элементы, которые меньше или равны опорному, перемещаются влево, элементы больше или равны — вправо. Затем процесс рекурсивно повторяется для левой и правой частей.

## Сложность
- Лучшая / Средняя: $O(n \log n)$ — когда pivot каждый раз делит массив примерно пополам.
- Худшая: $O(n^2)$ — когда массив уже отсортирован (или отсортирован в обратном порядке), а в качестве pivot выбирается крайний элемент.

## Применение
- `+`: Высокая скорость в среднем, работает «на месте» (in-place), не требуя выделения дополнительной памяти под элементы массива.

- `-`: Худший случай ужасен; неустойчива; рекурсивная реализация может привести к переполнению стека (поэтому на практике её часто делают итеративной со своим стеком)

## Устойчивость
Неустойчивая

## Реализация
Приведена в [quicksort.c](./quicksort.c)

```c
void quicksort(int *arr, int left, int right){
    if (left >= right) return;
    int pivot = arr[(left+right)/2];
    int i = left, j = right;
    while (i <= j){
        while (arr[i] < pivot) i++;
        while (arr[j] > pivot) j--;
        if (i <= j) {
            int temp = arr[i];
            arr[i] = arr[j];
            arr[j] = temp;
            i++, j--;
        }
    }
    quicksort(arr, left, j);
    quicksort(arr, i, right);
}
```
