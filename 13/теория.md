# Поиск путей в графе. Алгоритм Флойда-Уоршела

## Назначение

Нахождение кратчайших путей между всеми парами вершин графа одновременно.

## Принцип

Динамическое программирование. Пусть d[i][j] — длина кратчайшего пути из $i$ в $j$. Мы поочередно перебираем каждую вершину $k$ в качестве промежуточного пункта и проверяем: станет ли путь из $i$ в $j$ короче, если пройти его транзитом через $k$?
Формула: $d_{new}[i][j] = \min(d[i][j], d[i][k] + d[k][j])$.  

## Сложность

Лучшая / Худшая: Строго $\Theta(V^3)$ за счет трех вложенных циклов.

## Зачем нужен
- `+`: Умеет работать с отрицательными весами ребер.
- `+`: Сверхпростая реализация (всего 3 вложенных цикла).
- `+`: Позволяет легко обнаружить наличие в графе цикла отрицательного веса (если после работы алгоритма какой-то элемент на главной диагонали d[i][i] стал < 0).
- `-`: Дорогой по времени ($O(V^3)$), непригоден для графов, где количество вершин измеряется тысячами.

## Реализация
Приведена в [floyd_warshall.c](./floyd_warshall.c)
```c
#define V 6
#define INF 1000000

void floyd_warshall(int graph[V][V]){
    int d[V][V];
    for(int i = 0; i < V; ++i){
        for(int j = 0; j < V; ++j){
            d[i][j] = graph[i][j];
        }
    }
    for(int k = 0; k < V; ++k){
        for(int i = 0; i < V; ++i){
            for(int j = 0; j < V; ++j){
                if (d[i][k] != INF && d[k][j] != INF && d[i][k] + d[k][j] < d[i][j]) {
                    d[i][j] = d[i][k] + d[k][j];
                }
            }
        }
    }
    for(int i = 0; i < V; ++i){
        if(d[i][i] < 0){
            printf("Обраружен цикл с отрицательным весом");
        }
    }
}
```
