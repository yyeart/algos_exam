# Графы. Виды графов. Различные представления графов.
## Виды
Ориентированные (дуги имеют направления)/Неориентированные (простые связи); Взвешенные (у ребер есть вес)/Невзвешенные; ациклические (без циклов)

## Способы представления
1. Матрица смежности: Двумерный массив $V \times V$. `matrix[i][j]` хранит вес ребра
  - `+`: Проверка наличия ребра за $O(1)$
  - `-`: Есть $O(V^2)$ памяти, даже если ребер почти нет
2. Списки смежности: Массив списков размера $V$. Для каждой вершины хранится список связанных с ней вершин
   - `+`: Память $O(V + E)$ - идеально для разреженных графов
   - `-`: Проверка ребра между $i$ и $j$ требует обхода списка за $O(deg(i))$
3. Матрица инцидентности: Таблица $V \times E$ (вершины на ребра).
4. Список ребер: Массив структур, где каждая структура - одно ребро
    ```c
    struct Edge {
        int src;
        int dest;
        int weight;
    }
    ```
    - `+`: Идеально для алгоритмов, где нужно сортировать ребра по их весу
    - `-`: Неудобно искать соседей конкретной вершины
