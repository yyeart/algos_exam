# Алгоритмы поиска минимального остова графа
- Суть: найти связный подграф, содержащий все вершина исходного графа, без циклов, имеющий минимальный суммарный вес ребер. Он должен быть взвешенным и связным

## Алгоритм Прима
- Принцип: Растет из одной вершины. На каждом шаге ищется ребро минимального веса, один конец которого лежит в уже построенном дереве, а второй - снаружи. Вторая вершина добавляется в дерево.
- Сложность: $O(V^2)$ на матрице смежности; $O(E \log V)$ при использовании двоичной кучи (списки смежности).
- Псевдокод:
  ```
  1. Инициализировать MST пустой очередью
  2. Выбрать стартовую вершину v, установить ее метку dist[v] = 0. Для всех остальных dist[u] = INF
  3. Пока в MST не включены все вершины:
    a. Найти вершину u с минимальным dist[u] среди еще не включенных
    b. Найти u в MST
    c. Для каждого соседа w вершины u:
        если w не в MST и вес(u, w) < dist[w], то:
            dist[w] = вес(u, w)
            parent[w] = u
  ```
- Реализация
  Приведена в [prim,c](./prim.c)
  ```c
  #include <stdio.h>
  #include <stdlib.h>

  #define V 5
  #define INF 1000000

  int min_key(int key[], int mst_set[]){
      int min = INF, min_index = -1;
      for(int v = 0; v < V; ++v){
          if(!mst_set[v] && key[v] < min){
              min = key[v];
              min_index = v;
          }
      }
      return min_index;
  }

  void prim_mst(int graph[V][V]) {
      int parent[V];
      int key[V];
      int mst_set[V];
      for(int i = 0; i < V; ++i){
          key[i] = INF;
          mst_set[i] = 0;
      }
      key[0] = 0;
      parent[0] = -1;

      for(int count = 0; count < V - 1; count++) {
          int u = min_key(key, mst_set);
          if(u == -1) break;
          mst_set[u] = 1;
          for(int v = 0; v < V; ++v){
              if(graph[u][v] && !mst_set[v] && graph[u][v] < key[v]) {
                  parent[v] = u;
                  key[v] = graph[u][v];
              }
          }
      }
      for(int i = 1; i < V; ++i){
          printf("%d - %d \tВес: %d\n", parent[i], i, graph[i][parent[i]]);
      }
  }
  ```

## Алгоритм Краскала
- Принцип: Работает напрямую с ребрами. Сортируем все ребра графа по возрастанию веса. Идем по списку и берем ребро, если его добавление не создает цикл в уже собранном остове
- Сложность: $O(E \log E)$ или, что то же самое, $O(E \log V)$ — из-за базовой сортировки всех ребер.
- Псевдокод
  ```
  Краскал(Граф G):
    MST = пустой список ребер
    Для каждой вершины v из G->V:
      InitSet(v)
    Сортировка G->E по возростанию weight
    Для каждого ребра (u, v) из отсортированного G->E:
      Если FindSet(u) != FindSet(v):
        Добавить (u,v) в MST
        Объединить компоненты Set(u, v)
    Вернуть MST
  ```
