# Сортировка Шелла (Shellsort)

## Алгоритм

Модификация сортировки вставками. Сравниваются и меняются местами элементы, стоящие не рядом, а на расстоянии $k$ (шаг). На каждом шаге дистанция $k$ уменьшается (например, $k = n/2, n/4 ...$), пока не станет равной 1. Последний проход — это обычная сортировка вставками, но массив к этому моменту уже почти отсортирован, что делает её очень быстрой.

## Сложность

- Лучшая: $O(n \log n)$ или $O(n)$ в зависимости от выбранной последовательности шагов.
- Худшая: Зависит от последовательности шагов. Для классической формулы Шелла ($n/2^i$) худший случай — $O(n^2)$.

## Применение

- `+`: Не требует дополнительной памяти, намного эффективнее стандартного $O(n^2)$ при простом коде.
- `-`: Средняя сложность сильно зависит от выбора шагов; всё ещё медленнее, чем Quicksort или Heapsort на огромных массивах.

## Устойчивость

Неустойчивая

## Реализация

Приведена в [shellsort.c](./shellsort.c)
```c
void shellsort(int* arr, int n){
    for(int gap = n / 2; gap > 0; gap /= 2) {
        for(int i = gap; i < n; ++i){
            int temp = arr[i];
            int j;
            for(j = i; j >= gap && arr[j - gap] > temp; j -= gap) {
                arr[j] = arr[j - gap];
            }
            arr[j] = temp;
        }
    }
}
```
